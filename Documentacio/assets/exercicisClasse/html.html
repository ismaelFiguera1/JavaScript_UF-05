L'algoritme de css

L'algoritme CSS es la forma en que el navegador aplicara els estils als documents HTML. Es molt important entendre-ho per saber com s'apliquen i alguns casos com es sobreescriuen les regles CSS. Per entendre l'algorisme css cal tenis clars els seguents conceptes

  1. La cascada
  2. L'especifitat
  3. L'herencia

  1. La Cascada. Depen de :

L'origen del codi. Es d'on ve el codi. Hi ha tres possibilitats:

  1. User Agent
  2. Personalitzacions de l'usuari
  3. Estils d'autor

Especifitat

  1. Per cada etiqueta o pseudoelements suma 1 al 1r digit 0,0,0,1
  2. Classes, atributs o pseudoclasses suma 1 al 2n digit 0,0,1,0 per cada
  3. Identificador, suma 1 al 3r digit 0,1,0,0 per cada
  4. Suma 1 al 4rt digit 1,0,0,0

Tipus de caixes

  • En linea (Elements del formulari, span, p)
  • En block (div, section, tags semantics (nav, main, article, header, footer))

Les caixes en block

Es comporten de la seguent forma:

  • La caixa ocupa tota la linea i força un salt de linea al final
  • La caixa s'exten en la direccio de la linea fins al final. Generalment significa que la caixa tindra l'amplada del seu contenedor i omplira el 100% de l'espai disponible
  • Es respecten les propietats width i height
  • El padding, el border i el marge (Aquest ultim mante allunyats de a caixa la resta d'elements) s'apliquen

Les caixes el linea

Les caixes en linea es comporten de la seguent forma:


  • La caixa nomes ocupa l'espai que ocupa el seu contingut i no força un salt de linea.
  • Les propietats width i height NO s'apliquen.
  • El padding, el border i el margin Verticals s'apliquen, pero, NO mantenen allunyades altres caixes en linea
  • El padding, el border i el margin Horitzontals s'apliquen i mantenen allunyades altres caixes en linea

El posicionament

Les propietats FLOAT i POSITION ens permeten modificar la posici natural de qualsevol element del document HTML

El navegador posiciona cada element tenint en compte l'ordre en el que apareixen al document (esquerra a dreta i de dalt a baix) i del seu tipus de visualitzacio depenent de si es un element en linea o en block

amb FLOAT i POSITION podem modificar aquest comportament

Tipus de posicionaments

  • static (default)
  • relative
  • absolute
  • fixed
  • sticky

Per mourer i posicionar els elements amb relative, absolute, fixed o sticky utilitzarem les propietats:

  • top
  • bottom
  • left
  • right
  • z-index(profunditat, eix z)

Aquestes 5 propietat NO funcionen amb el valor per defecte static! Perque static significa respectar el flux original del posicionament dels elements

Les propietats top i left tenen prioritat respecte bottom i right respectivament. En cas de conflicte, entre aplicar top o bottom, s'aplicara top. En cas de conflicte entre aplicar left o right, s'aplicara left.

Posicionament static

En aquest posicionament NO s'apliquen left, right, top, bottom i z-index

Caixa estatica

Posicionament relative

Aquest posicionament l'element conserva les dimensions i el posicionament original dns el flux del document. Si es mou com a referencia la seva posicio original

Caixa relativa
Caixa absluta

L'element perd les seuves dimensions (width i height) i la posicio original al flux del document, si es mou, pot prendre cm a referencia 2 elements:

  1. El primer ancestre amb posicionament relatiu
  2. Si no troba cap ancestre amb posicionament relatiu, es mou respecte el document html
Caixa absoluta amb pare relatiu

Posicionament Fixed

Perd les dimensions i posicio original del flux del document. Si es mou pren com a referencia el document html i queda fix a la posicio on l'hem mogut. Quan hi hagi un scroll l'element no es moura, quedara fix.

Caixa fixa

Posicionament sticky

Es una combinacio de posicionament relatiu i fixed. En aquest posicionament les propietats top, left, bottom i right no mouen l'element, en comptes de mourel serveixen com a punt de referenciamentre l'element no arrivi als valors establerts. Mentre no s'arrivi a les posicions establertes per les propietats l'element es comportara com a relatiu i cuan arrivi a les posicionses convertira en fixed.

Hi ha 2 opcions d'implementaci:

  • L'element sticky ha de ser fill directe del body
  • El seu element contenedor ha de tenir dimensions definides i nomes serea sticky dins de les dimensions del contenedor
